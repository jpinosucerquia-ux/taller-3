Este proyecto implementa una aplicación en Python capaz de cargar archivos médicos en formato DICOM, extraer metadatos relevantes según el estándar, organizar los datos en un DataFrame de Pandas, calcular la intensidad promedio de cada imagen a partir del pixel_array y exportar los resultados a un archivo CSV. El proyecto simula una parte del flujo de un sistema PACS, similar a soluciones como dcm4chee, donde los estudios médicos se vuelven accesibles, consultables y estructurados. Además, se utiliza Programación Orientada a Objetos mediante la clase ProcesadorDICOM para encapsular la lógica de carga, extracción y análisis.

DICOM y HL7 son fundamentales para la interoperabilidad en salud porque permiten que diferentes sistemas hospitalarios intercambien información de manera estandarizada. DICOM es el estándar especializado en imágenes médicas y sus metadatos, definiendo cómo se almacenan y transmiten estudios como TC, RM o RX, además de asegurar compatibilidad entre equipos de distintos fabricantes. Por su parte, HL7 es un estándar orientado a la mensajería clínica y administrativa, permitiendo el intercambio de datos como información demográfica, órdenes médicas, resultados de laboratorio y movimientos del paciente. Conceptualmente, DICOM se centra en imágenes y su pixel data, mientras HL7 se enfoca en datos textuales y procesos administrativos. Ambos se complementan para lograr una verdadera interoperabilidad dentro de un hospital.

El análisis de la distribución de intensidades en una imagen médica tiene gran relevancia tanto clínica como computacional. Desde la perspectiva clínica, permite identificar distintos tipos de tejidos según sus rangos de intensidad, como hueso, grasa, aire o tejidos blandos, y facilita la detección de lesiones hipo o hiperintensas, hemorragias, tumores o edema. Es también un componente clave en el diagnóstico asistido por inteligencia artificial. Desde el punto de vista del preprocesamiento, estudiar las intensidades permite aplicar normalización, reducir variabilidad entre estudios, detectar valores atípicos o imágenes corruptas y optimizar procesos como segmentación, clasificación, filtrado y umbralización. El cálculo de la intensidad promedio es un primer paso simple pero útil para caracterizar el comportamiento general de la imagen.

Durante el desarrollo del proyecto se presentaron varias dificultades, como la presencia de archivos no válidos que debieron manejarse con bloques try/except, la ausencia de tags debido a procesos de anonimización que se solucionó usando getattr, las diferencias entre modalidades que afectan la estructura de pixel_array y la existencia de archivos sin datos de imagen, para los cuales se asignó un valor NaN en el cálculo del promedio. Las herramientas de Python fueron esenciales para superar estos retos: pydicom permitió leer y explorar los archivos DICOM sin necesidad de software especializado, numpy facilitó el manejo de matrices de imagen de forma eficiente, pandas permitió organizar los datos en tablas estructuradas y los entornos virtuales aseguraron una gestión adecuada de dependencias. Gracias a este ecosistema, el proyecto pudo implementarse de manera modular, clara y cercana a sistemas reales empleados en bioingeniería y procesamiento de imágenes médicas.
